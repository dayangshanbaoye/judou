# 03 · 开发路线图与里程碑

> 把《产品手册》的 MVP→v2 拆成**有序、可运行、可测试**的工作包。每个阶段产出一个能跑、能测的增量。验收标准一律写成「能被测试断言的事实」，直接作为 TDD 的起点。

ICAP 落地顺序：先跑通 **A/P**（读+讲解），再补 **C**（造句+TTS），再补复习（SRS），最后健壮性打磨。

---

## Phase 0 · Walking Skeleton（可运行骨架）
**目标**：端到端打通最小链路，CI 跑起来。
- E0.1 工程脚手架：Tauri2 + Vue3 + TS + SQLite，迁移运行器（`05_schema.sql` 空跑建库）。
- E0.2 一个 `ping` command + 前端调用 + 一条事件回传。
- E0.3 CI：`cargo test` + `cargo clippy -D warnings` + `pnpm test` 全绿。
- E0.4 注入式 `Clock` 与 `MockLlm`/`MockTts` 骨架就位。

**验收**：
- 启动 app 能调 `ping` 返回；migration 后 DB 含全部表（查 sqlite_master 断言表数）。
- CI 三项检查通过。

---

## Phase 1 · EPUB 解析与可追溯（A/P 基础）
**目标**：导入 EPUB → 目录树 + 段落 + 内容分类，**句子之前的全部追溯结构**先就位。
- E1.1 解包 + 定位 OPF（`container.xml`）。
- E1.2 解析 spine/manifest → 正确阅读顺序。
- E1.3 解析 nav.xhtml / toc.ncx → `toc_nodes` 树（含 href#anchor、level）。
- E1.4 内容分类初判：`introduction/preface/body/title_only/excluded`（依据 epub:type + 启发式）。
- E1.5 清洗与段落抽取：剥脚注/页眉/图注，留档；建段落追溯锚点（source_href+元素路径）。
- E1.6 落库 `books/toc_nodes/paragraphs`；导入解析报告数据。
- E1.7 `fixtures/epub/` 建初始语料：epub3 脚注、epub2 ncx、frontmatter/bodymatter、嵌套目录、title-only 节点各 1 个最小样本 + 黄金期望。

**验收（fixtures 驱动）**：
- 对每个 fixture，断言：toc 节点数/层级、阅读顺序、各节点 content_type 初判、段落数与归属。
- 脚注 fixture：句子层未生成但段落中脚注文本被剔除且原文留档可取。
- 任一段落可回溯到 book（外键+查询测试）。

---

## Phase 2 · 断句引擎 + 异常台账（A/P 基础）
**目标**：段落 → 句子（entry），可追溯，异常入台账。
- E2.1 规则分句器（pragmatic 思路）+ 缩写词典（Mr./e.g./U.S./人名首字母…）。
- E2.2 处理小数/序号/引号跨句/对话等规则；标注低置信段落。
- E2.3 落库 `sentences`（含 offset、segmentation_method、status）。
- E2.4 `processing_log` + `processing_rules` 引擎：异常登记、规则可启用/版本化。
- E2.5 句子可追溯到段/节点/书。

**验收**：
- 黄金句子集：对每个 fixture 段落，断言切出的句子序列==期望（用 `insta` 快照）。
- 缩写/小数/引号用例不误切（property/case 测试）。
- 触发的特殊处理在 `processing_log` 有对应记录；加一条规则后重跑，行为按规则变化。
- 断句前后文本拼接==原段落文本（守护「不改写原文」）。

---

## Phase 3 · 阅读器与精读（A/P 闭环前端）
**目标**：能读、能自测、能纠错。
- E3.1 三栏阅读器：目录树 / 句子流 / 上下文；面包屑。
- E3.2 连续阅读 + 逐句精读/自测两模式；句子状态可视。
- E3.3 多句框选；句子「合并/拆分」纠错 → 写回 + 入台账 + 可固化规则。
- E3.4 导入向导：解析报告页 + 范围确认页（勾选正文/序言/排除）。

**验收**：
- 组件测试：目录点击切换、状态切换（unread→understood）、面包屑正确。
- 合并/拆分后 `sentences` 与 `processing_log` 正确变更（集成测试）。
- 范围确认排除的节点不参与断句。

---

## Phase 4 · LLM 讲解 + 卡片 + 学习点（A→C 过渡）
**目标**：单句/多句送讲解，得到结构化四段 + 学习点，落卡片。
- E4.1 `LlmProvider` trait + OpenAI 兼容/Anthropic 适配 + `MockLlm`。
- E4.2 讲解提示词（`07`）+ JSON schema 校验 + 解析失败留 raw_json。
- E4.3 `cards`/`card_sentences`（支持多句）/`explanations`/`learning_points` 落库。
- E4.4 卡片 UI：四段展示、学习点列表、卡内追问线程（`card_threads`）。

**验收（全程 Mock，CI 不联网）**：
- 给定 MockLlm 返回的固定 JSON，断言：translation/plain_rephrase 入库、phrases 拆成 learning_points、grammar 存 grammar_json。
- 非法 JSON → 报错且 raw_json 留底，不静默吞。
- 多句选区生成单张卡片，card_sentences 顺序正确。

---

## Phase 5 · Create 造句 + TTS（C 层）
**目标**：每学习点造句 + 生成可下载/跟读音频。
- E5.1 造句提示词（`07`）：中文→英文呈现 + usage_note + 变体。
- E5.2 `created_sentences` 落库；学习点可多句。
- E5.3 `TtsProvider` trait + 适配 + `MockTts`；`audio_assets`（含 word_timings_json）。
- E5.4 UI：C 面板（中文输入→英文→变体→生成音频→播放/下载）。
- E5.5 `promoted` 闸门：手动选择升级的学习点/造句。

**验收**：
- MockLlm 造句 → en_output/usage_note 入库；一个学习点可挂多条造句。
- MockTts → audio_assets 行写入、文件路径存在、word_timings 解析正确。
- 未 promoted 的学习点不进入复习候选。

---

## Phase 6 · FSRS 复习引擎（SRS）
**目标**：间隔复习闭环 + 启动提醒。
- E6.1 `fsrs-rs` 封装：rating(Again/Hard/Good/Easy)→新状态/ due；注入 Clock。
- E6.2 `srs_cards`/`review_logs`/`decks`（desired_retention、每日上限）。
- E6.3 启动队列：`due<=now AND NOT suspended`，受每日上限约束。
- E6.4 复习会话 UI：呈现→揭示→4 档自评→音频跟读；统计页。
- E6.5（可选）积累日志后用优化器拟合个人参数。

**验收（固定时钟，确定性）**：
- 同一卡给定 rating 序列 → due/状态推进与期望一致。
- 队列只含到期未暂停项，且不超每日上限。
- review_logs 完整记录每次（供后续优化）。

---

## Phase 7 · 搜索 / 台账中心 / 设置 / 备份
- E7.1 FTS5 搜索：跨句子/讲解/学习点/造句/对话 + 筛选。
- E7.2 处理台账中心 + 规则库可视化（跳回原文、固化规则）。
- E7.3 设置：LLM/TTS 供应商与 Key（keyring）、提示词模板版本、复习参数。
- E7.4 备份/导出（SQLite 单文件 + 音频目录；JSON 导出）。

**验收**：
- 关键词命中跨实体结果正确（含中英文混排，见 `05` 分词说明）。
- 台账「升级为规则」后写入 processing_rules 且下次解析生效。
- 导出文件可在新库导入回灌（v2 起）后数据一致。

---

## Phase 8 · 健壮性与体验打磨
- E8.1 断句 LLM 兜底（低置信段落，仅返回边界，禁改写）。
- E8.2 词级时间戳卡拉OK跟读高亮。
- E8.3 讲解版本管理（换模型/提示词重讲、保留历史）。
- E8.4 真实书回归集：把踩过的坑沉淀为 fixtures，形成回归护城河。

**验收**：
- LLM 兜底结果文本与原文一致（不改写测试）。
- 回归 fixtures 全绿，新书问题先加 fixture 再修。

---

## 里程碑视图
| 里程碑 | 含阶段 | 可演示成果 |
|---|---|---|
| M1 能读会讲 | P0–P4 | 导入书→精读→单/多句结构化讲解+学习点 |
| M2 会造会读 | P5 | 学习点造句+TTS 跟读 |
| M3 会复习 | P6 | FSRS 间隔复习+启动提醒 |
| M4 好用耐用 | P7–P8 | 搜索/台账/规则库/健壮性 |

> 每阶段结束跑全量测试 + clippy；不绿不进下一阶段。
